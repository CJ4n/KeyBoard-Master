Windows API - lab part

Keyboard Master

  • Example application
  • Task description:
    • Your task is to create a clone keyboard master application.
    • Inside the main window, there are falling down black square windows (25x25 px) with different speed.
    • New squares appear randomly at the top edge of the main window with a delay between 0.3 and 1.0s.
    • When the square touches the bottom edge of the window it disappears. There is a limit of 100 squares to be visible at once.
    • When SPACE button is pressed the most bottom square disappears.
    • The main window has a proper title and at startup is centred and its size is half of the screen size.
    • Use timers to animate squares.

    • Attention: Please do not use GDI in this part of task. All objects should be WinAPI windows.

  • Additional requirements:
    • on the taskbar there is only one window
    • combination Alt + F4 closes the application
    • show system menu when you click Alt + Spacebar, or right-click on a window in the taskbar
    • no flickering or blinking and properly repainted windows after moving
  • Tips:
    • GetWindowRect(), GetClientRect(), MoveWindow(), ShowWindow()
    • SetTimer(), KillTimer(), WM_TIMER
    • WS_CHILDWINDOW, WS_CLIPCHILDREN
    • InvalidateRect()
    • SetWindowLongPtr(), GetWindowLongPtr(), GWLP_USERDATA
  • Note: In all doubtful and untold aspects application should behave like an example app.
  • Scoring:
    • correctly displaying main window (title, size and position): 2 points
    • displaying child windows (black squares) in main windows: 2 points
    • falling down animation: 2 points
    • new squares appear randomly at the top edge: 2 points
    • removing the most bottom square by space: 2 points
    • removing squares at bottom edge: 2 points
    • Note: It is not possible to obtain points for incomplete functionality.
