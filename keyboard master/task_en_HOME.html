  • Task description:
    • Finish app implementation.

    • Attention: Please use GDI in this part of task only for painting windows background. All squares should still be WinAPI windows.

  • Functionality:
    • Add New game and Pause to the menu with proper shortcuts.
    • Squares should contain one letter centred inside of the window.
    • When a key with a letter is pressed the most bottom square with that letter disappear.
    • In the windown title, there are two counters for missed keys and the wrong key pressed.
    • The main window can not be resized now and is always the topmost.
    • When not active the game is paused.
    • Add context menu for changing background with color or bitmap (tiled by default or stretched).
    • The default background color is the same as the system color of the inactive window caption.
  • Additional requirements:
    • all of the functionality from the lab must still be working.
    • on the taskbar there is only one window
    • combination Alt + F4 closes the application
    • show system menu when you click Alt + Spacebar, or right-click on a window in the taskbar
    • no flickering or blinking and properly repainted windows after moving
    • application must work stable - it can not crash after some time
  • Tips:
    • Use double buffering for drawing with GDI.
    • Remember about proper memory management - remove all created objects.
  • Note: In all doubtful and untold aspects application should behave like an example app.

    • Pause at focus change: 1 point
    • Changing background with color (and proper default color): 2 points
    • Changing background with bitmap: 2 points
    • Changing bitmap mode: 1 point
